Applied its security fixes: open-redirect hardening against backslash smuggling on `?next=`, `quote()` on the next-path, cookie-flag match on delete, fallback-logging on post-change re-login.

### What I learned
- `hatch.force-include` is the right hammer for shipping non-Python assets inside a `packages = [...]` wheel — `-include` alone doesn't grab sibling dirs. Moved `templates/` and `static/` INSIDE `web_service/` so `Path(__file__).parent` resolves correctly post-install.
- The auth JWT didn't carry an email claim before. Needed one for the post-password-change re-login (auth revokes all sessions on change, so the web service has to log in again) and for rendering the user's email in the nav without an extra `/auth/me` hop that password-change-only tokens can't make anyway. Additive + defaulted so no existing caller breaks.
- Ruff's B008 flags `Depends(...)` as a default argument on every FastAPI handler. The repo already has per-file ignores for auth/ingest; added the web service's `main.py` and `deps.py` to the list.
